# Create Plan Command

사용자의 요청에 대해 체계적인 구현 계획을 작성합니다.

## 수행 단계

1. **요구사항 분석**
   - 사용자 요청의 핵심 목표 파악
   - 명확하지 않은 부분은 질문으로 확인

2. **현재 상태 파악**
   - 관련 코드/파일 탐색
   - 기존 구현 패턴 확인
   - 영향받는 컴포넌트 식별

3. **구현 계획 작성**
   다음 형식으로 `.claude/plan.md`에 작성:

   ```markdown
   # [기능/작업명] 구현 계획

   ## 목표
   - 핵심 목표 요약

   ## 현재 상태
   - 관련 파일 및 현재 구현 상태

   ## 구현 단계
   1. [ ] 단계 1: 설명
      - 상세 작업 내용
      - 관련 파일: `path/to/file.ts`
   2. [ ] 단계 2: 설명
      ...

   ## 고려사항
   - 잠재적 이슈
   - 의존성
   - 테스트 필요 항목

   ## 예상 영향
   - 영향받는 기능/페이지
   ```

4. **사용자 확인**
   - 계획을 요약하여 제시
   - 수정 또는 승인 요청

## 참고사항
- 시간 추정은 하지 않음 (구체적 단계만 제시)
- 각 단계는 독립적으로 검증 가능하게 분리
- 복잡한 작업은 더 작은 단계로 분할
