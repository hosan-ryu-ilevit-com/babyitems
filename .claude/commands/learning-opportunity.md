# Learning Opportunity Command

버그 수정이나 기능 개발 후, 이를 학습 기회로 전환하여 기록합니다.

## 수행 단계

1. **작업 내용 분석**
   - 최근 git diff 또는 커밋 확인
   - 어떤 문제를 해결했는지 파악
   - 어떤 접근 방식을 사용했는지 정리

2. **학습 포인트 추출**
   다음 항목들을 분석:
   - **근본 원인**: 왜 이 버그/이슈가 발생했는가?
   - **해결 방법**: 어떻게 해결했는가?
   - **배운 점**: 이 경험에서 무엇을 배웠는가?
   - **예방법**: 유사한 문제를 어떻게 예방할 수 있는가?

3. **학습 기록 작성**
   `.claude/learnings.md` 파일에 다음 형식으로 추가:

   ```markdown
   ## [날짜] [제목]

   ### 문제 상황
   - 어떤 증상이 있었는가
   - 어떤 상황에서 발생했는가

   ### 근본 원인
   - 왜 발생했는가 (기술적 원인)
   - 왜 놓쳤는가 (프로세스 원인)

   ### 해결 방법
   - 적용한 수정 내용
   - 관련 파일: `path/to/file.ts:line`

   ### 배운 점
   - 기술적 인사이트
   - 패턴/안티패턴

   ### 재발 방지
   - [ ] 체크리스트 또는 가이드라인 추가
   - [ ] 코드 리뷰 시 확인 포인트

   ### 관련 키워드
   `#keyword1` `#keyword2`
   ```

4. **요약 제시**
   - 핵심 학습 포인트 3줄 요약
   - 향후 참고할 수 있는 액션 아이템

## 활용 예시

**버그 수정 후:**
```
/learning-opportunity
```
→ 방금 수정한 버그에서 학습 포인트 추출

**특정 커밋 분석:**
```
/learning-opportunity [commit-hash]
```
→ 해당 커밋의 변경사항 분석

## 참고사항
- 같은 실수를 반복하지 않기 위한 기록
- 팀 또는 미래의 나를 위한 지식 축적
- 정기적으로 learnings.md 리뷰 권장
